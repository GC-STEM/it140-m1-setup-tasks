# IT 140 Codio Virtual Desktop Guide

Welcome to the **IT 140 Codio Virtual Desktop (CVD)**! The CVD contains the software needed to complete all course activities.

This **Guide** appears on the right, and the **Virtual Desktop** appears on the left. Collapse or expand this Guide as needed.

## First-Time Setup

Complete these steps the first time you use the CVD and after using **RESET VM**. Have your account credentials ready if you plan to use GitHub, OneDrive, or Google Workspace to store your work.

1. [ ] Select the **Web Browser** icon on the CVD taskbar.
2. [ ] Point the browser to https://github.com/GC-STEM/it140
3. [ ] Select **Setup Tasks | Codio Virtual Desktop**.
4. [ ] Follow the instructions in the activity’s **README.md** file.

> 💡 **Pro Tip:** Use the **Full Screen** button on the noVNC control bar when you need more desktop space. Exit full-screen mode to view the Guide and CVD side by side.

## Completing Course Activities

1. [ ] Select the **Web Browser** icon on the CVD taskbar.
2. [ ] Point the browser to https://github.com/GC-STEM/it140
3. [ ] Select the activity you want to complete.
4. [ ] Follow the instructions in the activity’s **README.md** file.

Save your work often. Keep a backup of important files in GitHub, cloud storage, or on your computer.

## Codio Browser Window

You will complete most tasks inside the CVD. You do not need to learn or use most other Codio features. The ones you may need from time to time are explained below.

### Guide Tab

The **Guide** tab appears on the right and displays these instructions.

- Collapse or expand the Guide as needed.
- Read the complete Guide at least once.
- **Mark as Complete** is optional and does not affect your grade.

### VM Tab

The **VM** tab appears on the left and contains the CVD where you will complete course activities.

#### noVNC Control Bar

The noVNC control bar handle (**>** or **<**) appears along the far-left edge of the CVD. Select the handle to show or hide the control bar.

Most students will use only **Full Screen**.

- **Show keyboard:** Opens an on-screen keyboard. This is mainly useful on tablets and other touch-screen devices.
- **Show extra keys:** Provides keys such as **Ctrl**, **Alt**, **Tab**, and **Esc** when your browser or device does not send them correctly to the CVD.
- **Full screen:** Expands the CVD to fill your screen. Select it again or press **Esc** to exit full-screen mode.
- **Settings:** Changes display and connection settings. Keep the default settings unless Codio Support instructs you to change them.
- **Disconnect:** Ends your browser’s connection to the CVD. It does not reset the VM or delete saved files. Reopen the VM tab to reconnect.

#### Virtual Machine Control Bar

The **Virtual Machine Control Bar** appears at the top of the VM tab. It displays the VM’s status and contains the **RESTART VM** and **RESET VM** buttons.

##### VM Status

- **Virtual Machine is Ready:** The CVD is ready to use.
- **Virtual Machine is Inactive:** Refresh the browser window and wait for the VM to reconnect. Do not select **RESTART VM** or **RESET VM** unless the VM remains unavailable.

##### RESTART VM

> ⚠️ **Warning:** Restarting closes all applications and may delete unsaved work.

Use **RESTART VM** only when the CVD remains frozen or unresponsive after you refresh the browser.

Restarting the VM:

- closes all open CVD applications;
- restarts the operating system; and
- keeps files and configuration changes that were already saved.

Save your work before restarting.

##### RESET VM

> 🚨 **Danger:** Resetting permanently deletes your files and configuration changes from the CVD. This action cannot be undone.

Use **RESET VM** only when your instructor or technical support directs you to do so.

Before resetting:

1. Back up important files to GitHub, cloud storage, or your computer.
2. Confirm that the backup contains your latest work.

After resetting, you must repeat the steps under **First-Time Setup**.

### Codio Menu Bar

You normally will not use the Codio menu bar. Complete your programming and file-management tasks inside the CVD.

These menu options may be helpful:

- **Tools → Virtual Machine → Open:** Reopens the VM tab if it is missing or was accidentally closed.
- **Help → Help Center:** Opens Codio’s documentation.
- **Help → System Status:** Shows whether Codio is experiencing an outage or service disruption.
- **Help → Support:** Opens a chat with **Codio Support**. It does not connect you to the SNHU IT Service Desk.

## Getting Help

Contact the appropriate support resource:

- **Codio Support:** Problems with Codio, the VM tab, or the CVD connection
- **SNHU IT Service Desk:** Problems with your SNHU account, authentication, Brightspace, or other SNHU systems
- **Your instructor:** Questions about assignments, course requirements, or Python code. Unless the matter is private, post your question in the General Questions discussion forum so other students can benefit from the answer.
